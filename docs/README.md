# «Топливный Дозор»

Карта топливной ситуации, которую заполняют сами водители через Telegram-бота: где нет 92-го, где
лимит на 95-й, а где торгуют из канистр. Каждое сообщение проверяется и попадает на общую карту.

## Проблема

На АЗС периодически пропадает топливо, вводятся лимиты отпуска «в одни руки», продавцы завышают
цены сверх стелы, а где-то торгуют из канистр без лицензии, разбавляют топливо или обманывают
на объёме. Официальная информация о реальной ситуации на конкретной заправке до водителя обычно
доходит с опозданием (если доходит вообще) — приходится проверять на месте, теряя время и топливо
в баке, которое и так может быть на исходе.

## Решение

«Топливный Дозор» — краудсорсинговый сервис: сами водители сообщают о ситуации на АЗС через
Telegram-бота (пропало топливо, лимит, очередь, завышенная цена, незаконная торговля, недолив,
контрафакт), а сервис проверяет обращения (антиспам, зона покрытия, дедупликация, сверка
метаданных фото) и показывает их на общей карте в реальном времени. Чувствительные категории
(незаконная торговля, контрафакт, мошенничество) проходят ручную модерацию перед публикацией.

## Какую помощь это должно оказать

- **Водителю** — до заезда на заправку видно на карте, есть ли смысл ехать именно туда: топливо
  в наличии, действует лимит, очередь или цена аномально высокая.
- **Сообществу** — коллективно фиксируются случаи незаконной торговли, недолива и контрафакта,
  которые иначе остаются известны только тем, кто лично столкнулся.
- **Модератору/оператору** — единая очередь обращений, требующих проверки, вместо разрозненных
  сигналов в чатах и соцсетях.

## Состав репозитория

| Каталог | Назначение |
|---|---|
| `backend/` | FastAPI-бэкенд: приём и выдача обращений, модерация, справочники |
| `bot/` | Telegram-бот на aiogram — единственный канал приёма обращений от водителей |
| `frontend/` | React+TypeScript+Vite веб-карта и панель модерации |
| `backend-rust/` | Экспериментальный Rust-бэкенд (не задействован в основном стеке) |
| `deploy/` | Дополнительный docker-compose для окружения деплоя |
| `docker-compose.yml` | Локальный/продовый запуск: PostGIS, Redis, MinIO, API, бот |
